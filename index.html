터미널 여는 법 ⌘ Cmd + Space → Spotlight 검색창에 터미널 → Return
  1. 1 Git

    설치 창이 열리면 '설치'를 누릅니다.

    xcode-select --install
  2. 2 Node.js LTS

    설치 창이 열립니다. 계속 '계속'을 눌러 완료하세요.

    PKG_URL=$(curl -fsSL https://nodejs.org/dist/index.json | ruby -rjson -e 'd=JSON.parse(STDIN.read); v=d.find{|x|x["lts"]}["version"]; puts "https://nodejs.org/dist/#{v}/node-#{v}.pkg"') curl -fsSL "$PKG_URL" -o /tmp/node-lts.pkg && open /tmp/node-lts.pkg
  3. 3 GitHub CLI

    설치 창이 열립니다. '계속'을 눌러 완료하세요.

    URL=$(curl -fsSL https://api.github.com/repos/cli/cli/releases/latest | ruby -rjson -e 'r=JSON.parse(STDIN.read); a=r["assets"].find{|x|x["name"].end_with?("_macOS_universal.pkg")}; abort("pkg 없음") unless a; puts a["browser_download_url"]') curl -fsSL "$URL" -o /tmp/gh.pkg && open /tmp/gh.pkg
  4. 4 Vercel CLI

    위 설치가 모두 끝난 후 터미널을 닫고 다시 여세요.

    npm i -g vercel

3단계. 로그인

GitHub 로그인 후 Git 설정과 Vercel 로그인까지 자동으로 합니다.

계속 터미널에서 실행합니다.

여기서부터 Git Bash 사용 시작 메뉴 검색창에 Git Bash → 실행 (Git 설치 시 자동으로 생깁니다)
gh auth login git config --global user.name "$(gh api user --template '{{if .name}}{{.name}}{{else}}{{.login}}{{end}}')" EMAIL="$(gh api user/emails -q '.[] | select(.primary) | .email' | head -n1)" if [ -z "$EMAIL" ]; then EMAIL="$(gh api user -q .email)" fi if [ -z "$EMAIL" ]; then EMAIL="$(gh api user --template '{{.id}}+{{.login}}@users.noreply.github.com')" fi git config --global user.email "$EMAIL" vercel login

당일에 할 것

Claude 앱을 열고, 아래 순서대로 프롬프트를 복사해 붙여 넣으면 됩니다.

1부. Code 탭

  1. Claude 앱을 열고 상단의 Code 탭을 클릭합니다.
  2. 시작 화면에서 Local을 선택합니다. (내 컴퓨터 폴더에 직접 접근하는 방식)
  3. 작업할 폴더를 선택하고, 사용할 모델을 고릅니다.
  4. Claude 앱 설정을 변경합니다. 앱 메뉴에서 설정을 열고 Claude Code를 클릭한 뒤:
    1. 권한 우회 모드 → ON
    2. 미리보기 → ON
    3. 미리보기 세션 유지 → ON
  5. 채팅창 아래에서 Auto-approve(편집 자동 승인)를 켭니다. (파일 수정·터미널 명령을 자동 실행)
  6. 이후 아래 프롬프트를 순서대로 복사해 붙여 넣으면 됩니다.

3부. GitHub — 비공개 저장소

아래 프롬프트를 Code 탭에 붙여 넣으면 비공개 GitHub 저장소를 자동으로 만들고 첫 푸시까지 완료합니다.

이 폴더를 git 저장소로 준비하고, GitHub CLI(gh)로 비공개 원격 저장소를 만들어 연결한 뒤 첫 푸시까지 해 줘. gh repo create 매뉴얼의 --private, --source, --push 를 참고해. gh는 터미널에서 이미 설치·로그인해 둔 상태야.

앱에서 승인 요청이 오면 허용하세요. 브라우저 로그인 창이 뜰 수 있습니다.

4부. Vercel — 연결·배포

3부에서 GitHub 저장소를 만든 뒤 진행합니다. 아래 프롬프트로 Vercel에 연결하고 배포까지 완료합니다.

Vercel CLI로 이 폴더를 진행해 줘. vercel은 터미널에서 이미 설치·로그인해 둔 상태야. vercel link로 프로젝트 연결하고, GitHub 저장소와 연결하려면 vercel git connect 문서를 보고 맞는 단계 실행해. 배포는 vercel deploy 문서대로 하고, 프로덕션은 --prod 옵션 안내에 따라 진행해.

진행 중 질문이 오면 답하고, 승인 요청이 오면 허용하세요.
