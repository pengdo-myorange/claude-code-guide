🔥 지옥의 AI 캠프 — 설정 가이드

캠프 전에 미리 해올 것을 완료해 오세요. 당일에 할 것은 캠프에서 함께 진행합니다.

사용 중인 OS

미리 해올 것

1단계. 가입 + Git 설치

  1. GitHub 가입/로그인 — 코드를 저장하고 관리하는 곳
  2. Git for Windows 다운로드 → 설치 시 모든 질문에 기본값으로 진행
  3. Vercel 가입/로그인 — 만든 것을 인터넷에 공개하는 곳

1단계. 가입

  1. GitHub 가입/로그인 — 코드를 저장하고 관리하는 곳
  2. Vercel 가입/로그인 — 만든 것을 인터넷에 공개하는 곳

2단계. Claude Code 설정

  1. Claude 데스크톱을 다운로드·설치한 뒤 로그인합니다.
  2. Claude 앱을 열고 상단의 Code 탭을 클릭합니다.
  3. 시작 화면에서 Local을 선택합니다. (내 컴퓨터 폴더에 직접 접근하는 방식)
  4. 작업할 폴더를 선택(새 폴더 생성)하고, 사용할 모델을 고릅니다. (Sonnet 4.6)

    주의: 구글 드라이브, 원드라이브, iCloud Drive, Dropbox 등 클라우드 동기화 폴더는 제외하세요.

  5. Claude 앱 설정을 변경합니다. 앱 메뉴에서 설정을 열고 Claude Code를 클릭한 뒤:
    1. 권한 우회 모드 → ON
    2. 미리보기 → ON
    3. 미리보기 세션 유지 → ON
  6. 채팅창 아래에서 권한 건너뛰기를 켭니다. (파일 수정·터미널 명령을 자동 실행)

3단계. 채팅창에서 설치

아래 프롬프트를 순서대로 Claude Code 채팅창에 입력합니다.

  1. 1 Node.js

    Node.js는 다양한 개발 도구를 실행하는 데 필요한 기본 엔진입니다. Vercel CLI 등이 이 위에서 동작합니다.

    node.js를 자동 설치하고 PATH를 새로고침해

    권한을 요청하면 Always allow for this project를 클릭합니다.

    완료 후 Claude 앱을 재시작합니다. 작업표시줄에서 Claude 아이콘을 우클릭 → 창 닫기(또는 종료)로 완전히 종료한 뒤 다시 실행하세요.

  2. 2 GitHub CLI + 로그인

    CLI(Command Line Interface)는 버튼을 클릭하는 대신 텍스트 명령어로 프로그램을 조작하는 방식입니다. GitHub CLI를 설치하면 브라우저 없이도 저장소 생성·코드 업로드 등을 할 수 있습니다.

    github cli를 자동 설치하고 로그인해

    터미널에서 HTTPS / SSH를 물어보면 HTTPS를 선택합니다.

  3. 3 Vercel CLI + 로그인
    vercel cli를 자동 설치하고 로그인해

    설치가 끝난 뒤 Vercel plugin for Claude Code를 설치하겠냐고 물어보면 OK로 진행합니다.

아이디어 숙제

바이브 코딩으로 무엇을 만들지 3개 생각해 오세요. 실제 업무에 바로 쓸 수 있는 것이 좋습니다. 아이디어에 제한을 두지 마세요 — 뭐든지 가능합니다.

아이디어마다 아래 항목을 간단히 메모해 두면 캠프에서 바로 시작할 수 있습니다.

  • 만들고 싶은 것 — 한 줄 요약 (예: 팀 회의록 자동 정리 도구)
  • 누가 쓰나 — 사용자/대상 (예: 우리 팀 PM 5명)
  • 지금은 어떻게 하고 있나 — 현재 방식 (예: 회의 끝나면 수동으로 노션에 정리)
  • 참고할 만한 레퍼런스 — 비슷한 기성 앱/서비스
✓ 여기까지 완료하면 사전 준비 끝!

당일에 할 것

Claude Code 채팅창에 아래 프롬프트를 순서대로 복사해 붙여 넣으면 됩니다.

1부. 스킬 설치

스킬은 Code 탭에서 /로 쓰는 슬래시 명령 모음입니다. 아래 프롬프트를 순서대로 붙여 넣으세요.

gstack

Y Combinator 대표 Garry Tan이 만든 개발 워크플로 스킬. 코드 리뷰, QA 테스트, 배포, 디자인 검토 등을 슬래시 명령 하나로 실행합니다. 개발 전반에 걸쳐 사용합니다.

https://github.com/garrytan/gstack 스킬을 설치해

pm-skills

PM(프로덕트 매니저) 실무에 특화된 스킬 모음. 시장 조사, 경쟁사 분석, 사용자 페르소나, PRD 작성, OKR 설정 등 전략·리서치·실행 도구를 제공합니다. 기획 단계에서 활용합니다.

https://github.com/phuryn/pm-skills 스킬을 설치해

agent-skills

Google Chrome 팀의 Addy Osmani가 만든 프로덕션급 엔지니어링 스킬. 성능 최적화, 접근성 점검, 보안 감사, 테스트 작성 등 코드 품질을 높이는 데 사용합니다.

https://github.com/addyosmani/agent-skills 스킬을 설치해

2부. 아이디어 확정

사전 숙제로 생각해 온 아이디어 중 하나를 골라, 아래 템플릿을 채워서 /office-hours와 함께 붙여 넣으세요. /office-hours를 입력할 때 자동 완성되거나 글자 색상이 바뀌는지 확인하세요. 그렇지 않으면 1부에서 gstack 스킬이 제대로 설치되지 않은 것입니다.

/office-hours 만들고 싶은 것: 누가 쓰나: 지금은 어떻게 하고 있나: 참고할 만한 레퍼런스:

Claude가 질문을 하면 답하면서 아이디어를 다듬어 갑니다. 끝나면 프로젝트 이름과 방향이 정해집니다.

진행 팁
  • AI의 질문에 모두 답하려고 애쓸 필요 없습니다. 주도권은 내가 가지고, 필요 없어 보이는 질문은 건너뛰어도 됩니다.
  • 질문이 어려우면 "더 쉽게 설명해 줘"라고 다시 요청하세요.
  • Claude가 터미널 명령 실행 등 직접 할 수 있는 일을 내게 시키면 "네가 직접 해 줘"라고 답하세요. 대부분 바로 처리해 줍니다.

3부. GitHub + Vercel 배포

아래 프롬프트 하나로 비공개 GitHub 저장소 생성, 첫 푸시, Vercel 연결·배포까지 한 번에 진행합니다.

프로젝트 이름을 한국어로 정했다면 영어 이름도 함께 준비하세요. GitHub 저장소는 영어 이름으로 생성하도록 Claude에 요청하면 됩니다.

이 폴더를 git 저장소로 만들고, GitHub CLI로 비공개 저장소를 생성해서 첫 푸시까지 완료해. 그다음 Vercel CLI로 프로젝트를 연결하고 프로덕션 배포까지 진행해. gh와 vercel은 이미 설치·로그인된 상태야.

진행 중 질문이 오면 답하고, 승인 요청이 오면 허용하세요.
